# 测试方案及用例 (test_plan.md)

## 1. 测试目标
验证“高校固定资产全生命周期管理系统”的各项功能是否符合设计要求，重点确保**单一 Web URL 在 PC 与移动端浏览器下的响应式适配能力**、资产数据准确性以及业务流程连通性。

## 2. 测试策略
- **单元测试 (Unit Test)**：使用 JUnit 对后端核心算法（如折旧计算、二维码解析逻辑）进行验证。
- **集成测试 (Integration Test)**：测试前后端 RESTful 接口调用的一致性。
- **响应式测试 (Responsive Test)**：重点验证 UI 在 375px (iPhone)、768px (iPad) 和 1440px (PC) 宽度下的自适应显示。
- **摄像头调用测试 (WebRTC Test)**：验证在 HTTPS 环境下，不同品牌手机（iOS/Android）对 Web 摄像头的开启权限与对焦稳定性。

## 3. 测试用例 (核心流程)

### 3.1 资产入库与二维码生成
| ID | 测试场景 | 预期结果 |
| :--- | :--- | :--- |

| TC-02 | 二维码生成校验 | 资产详情页可实时生成并下载该资产的二维码 |

### 3.2 移动端扫码借用/归还 (WebRTC)
| ID | 测试场景 | 预期结果 |
| :--- | :--- | :--- |
| TC-03 | 移动端浏览器扫码识别 | 开启摄像头成功，扫码后自动填充资产详情 |
| TC-04 | 借用权限校验 | 系统根据当前登录账号权限，校验资产可借用状态 |
| TC-05 | 状态变更联动 | 扫码归还成功后，资产状态由“在用”变更为“闲置” |

### 3.3 报修与维护闭环
| ID | 测试场景 | 预期结果 |
| :--- | :--- | :--- |
| TC-06 | 移动端扫码报修 | 现场扫码并成功上传故障照片，后端记录流转日志 |
| TC-07 | 维修进度反馈 | 维修员更新状态，管理员可在 PC/移动端同步查看最新状态 |

### 3.4 响应式与分辨率兼容性
| ID | 测试场景 | 预期结果 |
| :--- | :--- | :--- |
| TC-08 | PC 模式 (>=1200px) | 侧边栏常驻，表格展示完整多列字段，图表完整平铺 |
| TC-09 | 移动模式 (<768px) | 侧边栏折叠/底部 Tab 激活，表格转换为列表卡片模式 |

## 4. 兼容性测试
- **环境要求**：生产环境必须部署 **HTTPS 协议**以保障 MediaDevices 接口调用。
- **浏览器兼容**：Chrome (主流), Safari (iOS 必须), 微信内置浏览器。
- **设备类型**：iPhone 12+, Huawei P40+, Samsung Galaxy, iPad, 宽屏 PC。
