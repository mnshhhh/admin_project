# 数据库设计 — 高校固定资产全生命周期管理系统

数据库：`asset_management`，字符集 `utf8mb4`，共 **15 张表**，分为三组：RBAC 权限模型（6 张）、业务表（8 张）、审计日志（1 张）。

### ER 关系汇总

| 关系 | 类型 | 说明 |
|------|------|------|
| sys_user → sys_dept | N:1 | 用户归属部门 |
| sys_user ↔ sys_role | N:N | 通过 `sys_user_role` 关联 |
| sys_role ↔ sys_menu | N:N | 通过 `sys_role_menu` 关联 |
| sys_dept → sys_dept | 自引用 | `parent_id` + `ancestors` 构建部门树 |
| sys_menu → sys_menu | 自引用 | `parent_id` 构建菜单树 |
| sys_asset → sys_dept | N:1 | 资产归属部门 |
| sys_asset → sys_user | N:1 | 资产责任人 |
| sys_asset → asset_category | N:1 | 资产分类 |
| asset_category → asset_category | 自引用 | `parent_id` 构建分类树 |
| asset_borrow → sys_asset | N:1 | 借用对应资产 |
| asset_borrow → sys_user | N:1 | 申请人 / 审批人 |
| asset_allocate → sys_asset | N:1 | 调拨对应资产 |
| asset_allocate → sys_dept | N:1 | 调出 / 调入部门 |
| asset_repair → sys_asset | N:1 | 报修对应资产 |
| asset_repair → sys_user | N:1 | 报修人 / 维修人 |
| asset_check_task → asset_check_detail | 1:N | 任务包含多条盘点明细 |
| audit_record → 业务表 | 多态 | `biz_type` + `biz_id` 关联借用/调拨/报修/报废 |

## 五、设计要点

### 数据权限

`sys_user.dept_id` 和 `sys_asset.dept_id` 是数据权限的核心字段。`sys_role.data_scope` 控制角色可见数据范围，后端 `@DataScope` AOP 切面根据当前用户角色，在查询 SQL 中动态拼接 WHERE 条件：

| data_scope | 含义 | 自动注入 SQL |
|------------|------|-------------|
| 1 | 全部数据 | 不拼接条件 |
| 2 | 本部门及下级 | `AND d.dept_id IN (SELECT dept_id FROM sys_dept WHERE FIND_IN_SET({current_dept_id}, ancestors))` |
| 3 | 仅本部门 | `AND d.dept_id = {current_dept_id}` |
| 4 | 仅本人 | `AND u.user_id = {current_user_id}` |

### 逻辑删除

除 `sys_menu`、`sys_role_menu`、`sys_user_role`、`sys_log`、`audit_record`、`asset_check_detail` 外，其余表均包含 `deleted` 字段，由 MyBatis-Plus `@TableLogic` 注解自动处理，查询时自动追加 `WHERE deleted = 0`。

### 树状结构

`sys_dept`、`sys_menu`、`asset_category` 均采用 `parent_id` 自引用实现树状结构。`sys_dept` 额外使用 `ancestors` 冗余字段（如 `0,1,2`）支持 `FIND_IN_SET` 快速查询所有下级部门。

### 通用审批

`audit_record` 通过 `biz_type`（业务类型）+ `biz_id`（业务主键）实现多态关联，统一承载借用审批、调拨审批、报修审批、报废审批等不同业务的审批记录，避免每种业务单独建审批表。

### 冗余字段

`asset_borrow`、`asset_allocate`、`asset_repair`、`asset_check_detail` 表中冗余了 `asset_code` 和 `asset_name`，用于列表查询时避免关联 `sys_asset` 表，提升查询性能。
