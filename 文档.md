## 6.2.1 系统管理模块

### 6.2.1.1 用户管理功能

#### （1）查看用户列表

**需求说明**
系统管理员可查看所有用户账号列表，支持按用户名/昵称关键字搜索、按账号状态（正常/停用）筛选。

**代码实现路径**

| 层级            | 文件路径                                                                           | 关键行号      |
| ------------- | ------------------------------------------------------------------------------ | --------- |
| 前端页面          | `frontend/src/views/system/user/index.vue`                                     | 第 1-115 行 |
| 前端 API        | `frontend/src/api/system.ts`                                                   | -         |
| 后端 Controller | `backend/src/main/java/com/university/asset/controller/SysUserController.java` | 第 23-30 行 |
| 后端 Service    | `backend/src/main/java/com/university/asset/service/SysUserService.java`       | -         |

**技术实现逻辑说明**

前端调用 `getUserList` API 并传入分页和筛选参数（keyword、status），后端 `SysUserController.list()` 接收 `BaseQueryDTO` 分页参数和 `keyword`/`status` 筛选条件后调用 `userService.getUserPage(query)` 进行数据查询。在 `SysUserService` 中通过 `LoginUser` 获取当前登录用户的 `deptId` 和 `dataScope` 信息，自动拼接 `dept_id IN (...)` 条件实现数据权限隔离，校级管理员可查看全部用户，部门管理员只能看到本部门用户数据。

**需求说明**
不同角色登录后，首页看到的内容不同——管理员看到统计卡片和待办审批，普通用户看到我的借用记录。

## 6.2.4 待办审批模块

### （1）采购申请单审批 / 采购单跟进

**需求说明**
资产管理员对待审批的采购申请单进行审批（通过/驳回），审批通过后进入采购流程（分发供应商采购），采购完成后提交入库。

**代码实现路径**

| 层级     | 文件路径   | 关键行号   |
| ------ | ------ | ------ |
| <br /> | <br /> | <br /> |

\| 前端 API | `frontend/src/api/audit.ts` | `getPendingApplications` / `approveApplication` |
\| 前端页面 | `frontend/src/views/asset/application/approval.vue` | 采购申请审批页面 |
\| 后端 Controller | `backend/src/main/java/com/university/asset/controller/AssetApplicationController.java` | 第 43-45 行（待审批）、第 55-58 行（审批）、第 63-65 行（采购分发）、第 113-116 行（入库审批） |

\| 入库页面 | `frontend/src/views/asset/application/warehouse.vue` | 入库单填写与审批 |

**技术实现逻辑说明**

采购申请审批页面调用 `getPendingApplications()` 发起 `GET /asset/application/pending` 请求获取待审批列表，管理员点击"通过"或"驳回"后调用 `approveApplication(row.id, { action, remark })` 发起 `POST /asset/application/{id}/approve` 请求，后端 `approveApplication(id, action, remark)` 判断审批结果——审批通过时将申请单状态改为 `APPROVED` 并记录审批人、时间、备注，写入 `audit_record` 表；审批驳回时将状态改为 `REJECTED` 并记录驳回原因。审批通过后，管理员点击"采购分发"调用 `createPurchaseOrder()` 生成采购单记录（填写外部供应商 `externalPartner`、联系方式 `contactInfo`、预计到货日期 `expectedDate` 等），同时将申请单状态更新为 `PROCUREMENT`。采购完成后，管理员在入库页面填写入库单（关联采购单 `purchaseOrderId`），调用 `PUT /asset/application/warehouse-entry/{id}/approve` 审批入库，入库审批通过后采购单状态改为 `COMPLETED`，申请单状态改为 `WAREHOUSED`，系统自动向申请人发送站内通知（写入 `sys_notification` 表），告知资产已到货请联系领取。

### （3）按部门盘点

**需求说明**
创建资产盘点任务，关联特定部门，通过扫码方式逐一对资产进行盘点，记录正常/盘亏/盘盈/待确认状态。

**代码实现路径**

| 层级            | 文件路径                                                                                              | 关键行号                                                                                        |
| ------------- | ------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| 前端页面          | `frontend/src/views/asset/check.vue`                                                              | 第 1-180 行（完整页面逻辑）                                                                           |
| 前端 API        | `frontend/src/api/check.ts`                                                                       | `getCheckTasks` / `createCheckTask` / `scanCheck` / `completeCheckTask` / `getCheckDetails` |
| 后端 Controller | `backend/src/main/java/com/university/asset/controller/AssetCheckController.java`                 | -                                                                                           |
| 后端 Service    | `backend/src/main/java/com/university/asset/service/AssetCheckService.java`                       | `createTask()` / `scanCheck()` / `completeTask()`                                           |
| 后端 Entity     | `backend/src/main/java/com/university/asset/entity/AssetCheckTask.java` / `AssetCheckDetail.java` | -                                                                                           |

**技术实现逻辑说明**

按部门盘点采用左右分栏布局：左侧为盘点任务列表，右侧为选中任务的资产明细。创建盘点任务时，调用 `createCheckTask({ deptId, taskName })` 生成编号（格式 `CHECK-YYYY-NNN`），系统自动将目标部门下的所有资产批量导入 `asset_check_detail` 明细表，每条明细初始 `checkResult='PENDING'`（待盘点）。扫码盘点时，调用 `scanCheck(taskId, assetCode)` 发起 `POST /asset/check/task/{taskId}/scan` 请求，按资产编码定位到对应明细记录，更新 `checkResult`（NORMAL=正常 / SURPLUS=盘盈 / DEFICIT=盘亏 / PENDING=待确认）、`checkerId`（盘点人）、`checkTime`（盘点时间）。完成盘点时，调用 `completeCheckTask(taskId)` 发起 `PUT /asset/check/task/{taskId}/complete` 请求，将任务状态改为 `COMPLETED`，并汇总统计正常、盘亏、盘盈数量写入任务表。右侧明细列表支持按盘点结果类型筛选展示。
