# 参考内容
## 软件架构
本系统采用经典的前后端分离架构，主要由五个层次构成软件架构。
1. **Web层 (展示层)**：前端采用 Vue 3 + Vite + Element Plus + Pinia + Axios 框架。负责业务数据的可视化展示、人机交互以及前端路由。
2. **控制层 (Controller Layer)**：后端基于 Spring MVC (Spring Boot)，通过 RESTful 接口接收前端请求。它负责请求的分发、参数校验以及统一的 JSON 响应格式（R.java）。
3. **业务逻辑层 (Service Layer)**：包含业务服务接口及实现类。它是系统的核心，负责复杂的业务规则处理、事务管理以及权限验证（Spring Security）。
4. **数据持久层 (Mapper Layer)**：采用现代化的 MyBatis-Plus 框架。通过 Mapper 接口和 XML 映射文件，实现与 MySQL 数据库的高效交互。
5. **实体层 (Entity Layer)**：定义了与 MySQL 数据库表结构对应的 Java 对象（POJO），作为数据在各层之间流转的载体。

数据库层面，系统使用 MySQL 存储业务数据，并引入 Redis 用于会话管理与缓存。
数据交互方面，Web层与后端各层之间通过标准的 JSON 格式进行数据交互。

# 开发环境
本系统采用前后端分离的架构模式，有效提升了开发效率和系统的可维护性 [13] ，前端基于 Vue.js 框架实现用户交互界面，后端基于 Spring Boot 框架构建 RESTful API，并通过 Axios 实现前后端数据通信。具体的软硬件环境配置如表 7.1 所示，前端配置如图 7.1 所示，后端配置如图 7.2 所示。

在开发模式和框架方面，前端基于 Vue.js 3.5.x 开发。Vue.js 是一款渐进式 JavaScript 框架，通过声明式渲染和组件化开发，能帮助开发者快速构建交互复杂的单页面应用程序。Vue 3 引入了 Composition API，显著提升了代码的可读性和逻辑重用性 [14] 。同时，系统使用 Vue Router 5.x 管理前端路由，并采用 Pinia 3.x 进行全局状态管理，以提供更现代且简洁的状态维护机制。

后端则是基于 Spring Boot 3.3.x 开发，Spring Boot 框架通过“约定优于配置”的理念极大地简化了 Java EE 应用的搭建过程，具有开箱即用、开发效率高、易于部署等优点 [15] 。系统整合了 Spring Security 框架实现精细化的权限控制，并使用 MyBatis-Plus 3.5.x 增强持久层开发，简化了数据库操作。通信方式上，前端通过 Axios 发起异步 HTTP 请求，后端接口返回标准的 JSON 格式数据。
